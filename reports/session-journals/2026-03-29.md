## 4. Challenges and Mitigations

| Challenge | Root cause | Mitigation | Prevention |
|-----------|-----------|------------|------------|
| GitHub Projects V2 GraphQL API for adding status options used wrong mutation name | API changed; `createProjectV2FieldOption` doesn't exist | Used `updateProjectV2Field` with `singleSelectOptions` array to replace all options at once | Use GraphQL introspection (`__type`) to verify input field names before calling mutations |
| Branch protection API rejected `-f` flag parameters | `gh api` `-f` flag sends strings, but API expects boolean/integer types | Switched to `--input -` with JSON heredoc to send properly typed values | Always use `--input` with JSON for complex API payloads |
| Worker bot account couldn't edit project board items | Project board owned by `rantoine19`, worker bot lacks project access | Switch to `rantoine19` account for board operations, then back to worker for git operations | Grant project collaborator access to worker bot, or accept account switching |
| `git checkout main` failed in worktree | Main branch already checked out in parent worktree | Created feature branch directly from current HEAD (which tracked main) | In worktrees, use `git fetch origin main` + branch from current commit instead of checking out main |
| Node.js `/dev/stdin` doesn't work on Windows | Windows doesn't have `/dev/stdin`; process substitution also fails | Write to temp file in working directory, pass as argument to node | On Windows, always use temp files instead of stdin piping for Node.js |
| Duplicate epics from earlier bootstrap run | Previous bootstrap created issues #5-#14, this session created #15-#33 | Closed all 13 duplicates (#1-#2, #4-#14) during backlog grooming | Check for existing issues before creating new ones during bootstrap |

## 5. Insights and Learnings

**Technical insights:**
- Claude Vision can replace dedicated OCR services (Textract) for medical document parsing, significantly simplifying the architecture — single AI provider for both parsing and chat.
- Supabase as a unified backend (DB + Auth + Storage) with HIPAA BAA reduces vendor management overhead compared to assembling separate AWS services.
- GitHub Projects V2 GraphQL API requires `updateProjectV2Field` with a full `singleSelectOptions` array to set status column options — it replaces all options, not appends.

**Process insights:**
- Bundling bootstrap docs with the first implementation PR is efficient — avoids a separate "docs only" PR cycle.
- The Agentic PRD Lite ticket format (4 Power Sections) produces high-quality tickets that are immediately actionable. All 13 feature tickets passed quality review without needing refinement.
- Backlog grooming right after bootstrap catches dependency ordering issues early (Ready column was in wrong order).

**Domain insights:**
- The competitive landscape for AI health literacy tools is heating up (Hathr AI, Patiently AI, RosettaMD, ChatGPT Health). HealthChat AI's differentiator is the combination of HIPAA compliance + employer wellness channel + 5th grade simplification.
- Three MVP features (health simplification, risk flagging, doctor questions) were initially marked P1 but are actually core to the value proposition — elevated to P0 during grooming.
